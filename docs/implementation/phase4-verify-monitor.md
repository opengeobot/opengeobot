## 1. 阶段目标

完成“执行后验证 + 周期监控 + 复盘学习 + 策略推荐”的运营闭环。

## 2. 模块落地清单

### 2.1 效果验证（6.7）

- 支持 Baseline 与 After 对比。
- 支持按内容版本、Git commit、PR 关联变更。
- 支持重复运行降低随机性。
- 支持生成可追溯效果报告。

### 2.2 监控与告警（6.8）

- 支持频率配置：日/周/月。
- 支持告警类型：
  - 可见性回退
  - 引用率下降
  - 负面信息增长
  - 错误信息增长
  - 竞品跃迁
  - 内容过期
- 支持处理链路：状态、负责人、处理记录、关闭原因。

### 2.3 周期复盘与自我进化（6.9）

- 周报/月报自动生成。
- 策略记忆库沉淀：
  - Playbook 类型
  - 执行对象
  - 影响提示词
  - 效果变化
  - 适用场景
- 推荐机制：
  - 高成功策略加权
  - 无效策略降权
  - 相似场景策略复用

## 3. 策略记忆最小模型

- `strategyId`
- `playbookType`
- `evidenceSet`
- `impactPrompts`
- `impactMetrics`
- `successRate`
- `reuseScope`
- `lastValidatedAt`

- [x] 告警规则与处理闭环定义完成

## 5. 验收证据要求

- 至少 1 份 Baseline/After 对比报告。
- 至少 1 条告警触发到关闭的闭环记录。
- 至少 1 轮“复盘 -> 策略推荐 -> 下轮执行建议”结果输出。
