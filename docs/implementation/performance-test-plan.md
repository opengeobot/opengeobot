# OpenGEO Bot V1 性能压测计划

<!--
作者：AxeXie
创建时间：2026-05-08 13:25:00
-->

## 1. 压测目标

- 验证 PRD `10.4` 负载目标：
  - 单项目 1000 条提示词
  - 3 个引擎并行评测
  - 每日一次监控任务

## 2. 压测场景

### 场景 A：Baseline Run 峰值并发

- 输入：1000 条提示词、3 引擎
- 目标：
  - Run 任务成功率 >= 95%
  - 95 分位任务时延在可接受范围内

### 场景 B：错误重试压力

- 注入 10% 引擎超时与 5% 限流错误
- 目标：
  - 重试后成功率 >= 90%
  - 不出现队列雪崩

### 场景 C：报告与监控并发

- 执行 Scheduled Run 同时生成周报
- 目标：
  - 关键接口无超时
  - 报告生成成功率 >= 98%

## 3. 观测指标

- Run 成功率
- 引擎调用成功率
- 平均与 95 分位时延
- 错误率与重试率
- 队列积压深度
- 资源使用率（CPU、内存、I/O）

## 4. 退出准则

- 所有关键场景达到目标阈值。
- 未出现不可恢复错误。
- 输出压测报告并完成评审。
