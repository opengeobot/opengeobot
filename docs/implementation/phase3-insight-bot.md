# Phase 3：解析、洞察与 Bot 工作台实施说明

## 1. 阶段目标

形成“回答解析 -> 机会识别 -> Playbook 生成 -> 人工审批”的可运营闭环。

## 2. 模块落地清单

### 2.1 回答解析与指标计算（6.4）

- 解析能力：
  - 提及识别（品牌、别名、产品、仓库、包名）
  - 引用提取（链接、脚注、段落、域名）
  - 位置识别（列表排序、首次出现位置）
  - 情感分类（正向/中性/负向）
  - 风险标签（错误/过期/安全/合规/竞品替代）
- 指标计算：
  - Mention Rate
  - Citation Rate
  - Average Position
  - Sentiment Score
  - Share of Voice
  - Citation Quality

### 2.2 洞察中心（6.5）

- 关键页面视图：
  - 项目总览趋势
  - 引擎/地区/语言维度筛选
  - 竞品对比矩阵
  - 引用来源库
  - Top 20 机会列表

### 2.3 GEO Bot 工作台（6.6）

- 输出能力：
  - 内容 Playbook
  - 结构化数据 Playbook
  - 开源项目 Playbook
  - Markdown 草稿 / PR 草稿
- 治理能力：
  - 证据绑定
  - 风险分级
  - 人工审批门禁
  - 回滚建议

## 3. 数据追溯约束

- 每个洞察必须关联：
  - 原始回答 `responseId`
  - 解析规则版本
  - 指标计算版本
  - 触发提示词集合
- 每个 Playbook 必须关联：
  - 至少 1 个机会点
  - 至少 1 组可验证证据

## 4. 阶段门禁检查单

- [x] 回答解析规则体系定义完成
- [x] 六类核心指标计算口径定义完成
- [x] 洞察视图与机会优先级输出定义完成
- [x] Bot 草稿与审批流程定义完成
- [x] 页面文案接入 i18n 约束定义完成

## 5. 验收证据要求

- 至少 20 个可解释机会点样例。
- 至少 5 个 Playbook 草稿样例。
- 至少 1 条“洞察 -> Playbook -> 人工审批”链路记录。
