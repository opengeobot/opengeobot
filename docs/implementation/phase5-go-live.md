## 1. 阶段目标

完成 MVP 上线前的性能、可靠性、合规和运营门禁，确保可稳定进入正式运行。

## 2. 模块落地清单

### 2.1 性能压测与容量验证

- 目标负载（来自 PRD 10.4）：
  - 单项目 1000 条提示词
  - 3 个引擎
  - 每日一次监控任务
- 压测维度：
  - Run 并发吞吐
  - 引擎失败重试影响
  - 解析队列积压
  - 报告生成时延

### 2.2 稳定性与故障演练

- 演练场景：
  - 引擎超时
  - 限流触发
  - 部分失败
  - 配置错误发布后快速回滚
  - 告警风暴抑制
- 目标：
  - 关键业务链路可恢复
  - trace 链路可完整定位问题

### 2.3 安全与合规审计

- 检查项：
  - 密钥管理与明文禁止
  - 敏感数据脱敏与阻断
  - 审计日志完整性
  - 数据保留策略执行
  - 引擎调用合规性标注

### 2.4 UAT 与灰度发布

- 白名单项目灰度上线。
- 监控核心 SLO：
  - Run 成功率
  - 报告生成成功率
  - 告警处理时延
- 设定回滚阈值并演练。

### 2.5 MVP 终验签收

- 对照 `docs/opengeobot-v1.md` 中 `11.1` 与 `13` 条款逐项验收。
- 产出验收签字记录并归档。
