# OpenGEO Bot V1 灰度上线 Runbook

<!--
作者：AxeXie
创建时间：2026-05-08 13:27:00
-->

## 1. 发布前检查

- [ ] 所有阶段门禁通过。
- [ ] MVP 终验清单完成签核。
- [ ] 回滚版本可用并完成演练。
- [ ] 告警与值班机制已启用。

## 2. 灰度发布步骤

1. 发布到预生产并完成冒烟验证。
2. 白名单项目 10% 灰度。
3. 观察 24 小时核心指标：
   - Run 成功率
   - 引擎错误率
   - 报告成功率
4. 若指标稳定，扩大到 50%。
5. 全量发布并冻结 24 小时。

## 3. 回滚策略

- 触发条件：
  - Run 失败率超过阈值
  - 错误率持续升高
  - 关键链路不可用
- 执行动作：
  - 回滚到上一个稳定版本
  - 清理异常任务
  - 启动问题复盘

## 4. 发布后复盘

- 记录上线窗口内异常事件与处理动作。
- 输出上线报告、风险项与改进计划。
