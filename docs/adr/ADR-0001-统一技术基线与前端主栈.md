# ADR-0001：统一技术基线与前端主栈

| 属性 | 内容 |
| --- | --- |
| 状态 | Accepted |
| 日期 | 2026-07-03 |
| 决策来源 | 用户对本次任务的明确架构要求 |
| 影响范围 | 全仓库、M0-M6、云端、前端、Agent、边缘、数据与运维 |
| 替代文档 | 上游 PDF 第 4、15 节中的多候选技术组合 |

## 背景

上游《一脑多控平台详细设计说明书 V1.0》保留了 Go、React、多个 Agent Runtime、Kafka/RabbitMQ、Prometheus、Elasticsearch、Milvus 等候选项。候选并列会让 AI 编程 IDE 在不同功能中自行选型，造成双栈页面、重复基础设施、契约漂移和无法复现的部署结果。

本次决策把候选架构收敛为唯一实施基线。PDF 继续作为业务目标来源；工程实现以本 ADR、AI 开发规范、实施蓝图和机器清单为准。

## 决策

| 层级 | 统一基线 | 实施解释 |
| --- | --- | --- |
| 云端后端 | Java 21 + Spring Boot 3.x + MyBatis-Plus / Python 3.12 | 管理类领域和平台公共能力使用 Java；QwenPaw Provider、MCP、机器人/仿真适配使用 Python |
| 前端 | Vue 3 + TypeScript + Vite + WebSocket | Vue 是仓库唯一默认主栈；React 只允许用于无法迁移的既有嵌入组件或隔离微前端，且必须另行 ADR；禁止同功能双栈实现 |
| Agent Runtime | QwenPaw | 通过平台 `AgentRuntimeProvider` 端口隔离；不得把 QwenPaw 内部 API 当作平台契约 |
| 工具协议 | MCP 优先，HTTP/gRPC 辅助 | MCP 只暴露高层应用用例；底层 ROS topic、厂商 SDK 和原始控制协议不得暴露给 Agent |
| 边缘运行时 | ROSClaw Edge Runtime | 负责 Physical AI 运行时集成；平台仍自有 Safety Gateway、Local Skill Executor 和审计边界 |
| ROS2 接入 | ROS2 DDS / rosbridge / Zenoh | ROS2 为主路径，Ubuntu 24.04 + ROS 2 Jazzy 为首个兼容基线 |
| ROS1 接入 | ros1_bridge / rosbridge_suite / 自研 ROS1 Gateway | 只作为隔离兼容路径，不进入云端领域模型 |
| 弱网通信 | Zenoh | 用于弱网、边缘分布式通信和 ROS2DDS 桥接，不替代业务幂等和持久事件 |
| 数据库 | PostgreSQL + TimescaleDB 可选 | PostgreSQL 保存业务事实；达到实测遥测阈值后才启用 TimescaleDB |
| 对象存储 | MinIO / S3 | 保存图片、视频、地图、日志包和 bag；数据库只保存对象元数据 |
| 向量库 | pgvector | 保存任务记忆、失败案例和场景语义向量 |
| 消息队列 | NATS + JetStream | 承载任务、状态、告警和 Trace 事件，采用至少一次投递与幂等消费 |
| 监控 | vmagent + VictoriaMetrics + Grafana | 指标采集、存储、展示和告警统一使用该链路 |
| 日志 | Vector + Loki + PostgreSQL 审计表 | 运行日志进入 Loki；关键业务与安全审计追加写入 PostgreSQL |
| 告警 | Grafana Alerting / 自研告警服务 | 自研服务只负责业务告警生命周期、通知可靠性和升级，不复制指标规则引擎 |

“Vue / React”在本项目中的含义是“Vue 主栈 + React 的受控例外兼容”，不是并行建设两套指挥端。任何 React 例外必须证明不能使用 Vue 实现、定义隔离边界、包体与安全影响、测试方式和退出计划。

## 外部组件契约门禁

QwenPaw、ROSClaw、ROS2/ROS1 bridge、Zenoh 和厂商 SDK 在实现前必须：

1. 在机器清单的外部契约登记项中标记锁定版本。
2. 保存官方来源、许可证、版本或提交、制品摘要及本地锁定参考。
3. 以契约测试验证实际入口；禁止根据示例或模型记忆编造 API。
4. 记录升级兼容、降级和回滚步骤。
5. 对 ROSClaw 标为 Experimental/Research 的能力默认关闭。

相关功能在外部契约仍为 `UNPINNED` 时不得进入 `IN_PROGRESS`。

## 兼容与迁移影响

- 不新增 Go 管理服务，也不并行引入 Kafka、RabbitMQ、Prometheus、Elasticsearch 或 Milvus。
- 既有非基线实现若未来进入仓库，必须先通过迁移 ADR；迁移完成前不能作为新功能模板。
- M2 只交付 ROS2 仿真与主路径适配，不要求真实硬件 HIL。
- ROS1、Unitree 和自研协议放在 M3 隔离兼容适配中，完成受控 HIL 后才可声明对应设备支持。
- TimescaleDB、自研告警服务和 React 都是有条件能力，不是默认创建的平行实现。

## 安全影响

- QwenPaw 和所有 Agent 输出继续按不可信提案处理。
- ROSClaw 不替代平台 Safety Gateway；云端、Agent 和管理员均不能覆盖边缘 `BLOCK` 或本地急停。
- MCP/HTTP/gRPC 只能调用已注册 Capability/Skill，不能形成到底层运动接口的旁路。
- 弱网和消息重试采用命令签名、期限、防重放、幂等与 fencing，不能假设“恰好一次”。

## 回滚方案

若基线组件无法满足经测试确认的硬性需求：

1. 保持现有安全链和契约不变，暂停受影响功能。
2. 提交新的 ADR，附基准数据、兼容矩阵、迁移成本和安全回归结果。
3. 通过 Provider/Port 替换实现，不直接修改领域模型。
4. 在仿真、双写/影子流量或隔离环境验证后分阶段切换。
5. 保留上一锁定版本、数据库前向修复方案和部署回滚制品。

不得以“临时跑通”为由在业务模块内引入第二套公共能力或旁路安全网关。
